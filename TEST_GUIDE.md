# How to Test MediScribe — Step by Step

This walks you through testing the whole app. No coding required.

---

## Step 0: Make Sure Both Servers Are Running

You need two terminals open:

**Terminal 1 (Backend):**
```bash
cd HOOHACKS
source .venv/bin/activate
python manage.py runserver 8000
```
You should see: `Listening on TCP address 127.0.0.1:8000`

**Terminal 2 (Frontend):**
```bash
cd HOOHACKS
npm run dev
```
You should see: `Local: http://localhost:5173/`

---

## Test 1: Check the Backend is Alive

Open this in your browser:

**http://localhost:8000/api/health/**

You should see:
```
{"status": "ok", "service": "VoiceBridge API"}
```

If you see this, the backend is working.

---

## Test 2: Open the App

Open **http://localhost:5173** in your browser.

You should see the MediScribe landing page.

---

## Test 3: Create a Session (Backend Only)

Open a third terminal and run:

```bash
curl -X POST http://localhost:8000/api/sessions/ \
  -H "Content-Type: application/json" \
  -d '{"provider_id": "test_doctor", "patient_language": "es"}'
```

You should get back something like:
```json
{
  "id": "some-uuid-here",
  "provider_id": "test_doctor",
  "patient_language": "es",
  "created_at": "...",
  "ended_at": null,
  "medical_summary": "",
  "messages": []
}
```

**Copy the `id` value** — you need it for the next tests.

---

## Test 4: Test the Real-Time Translation (The Big One)

This tests the full pipeline: WebSocket → Gemini → ElevenLabs.

Save this as `test_ws.py` in the HOOHACKS folder, then run it:

```bash
source .venv/bin/activate
python test_ws.py
```

It will:
1. Create a session
2. Connect via WebSocket
3. Send "How long have you had this headache?" as a doctor message
4. Print the Spanish translation from Gemini
5. Tell you if audio was generated by ElevenLabs

**Expected output:**
```
Creating session...
Session: <some-uuid>
Connecting to WebSocket...
Connected!
Sending doctor message: "How long have you had this headache?"
Waiting for translation...
---
RESULT:
  Doctor said:  How long have you had this headache?
  Translated:   ¿Desde cuándo tiene este dolor de cabeza?
  Audio:        YES (49100 chars of base64 MP3)
---
Test PASSED — Gemini + ElevenLabs both working!
```

If you see "Test PASSED" — everything works end to end.

---

## Test 5: End a Session and Get a Summary

Using the session ID from Test 3:

```bash
curl -X POST http://localhost:8000/api/sessions/YOUR_SESSION_ID/end/
```

Then get the summary:

```bash
curl http://localhost:8000/api/sessions/YOUR_SESSION_ID/summary/
```

You should see a structured medical summary generated by Gemini.

---

## What If Something Fails?

| Problem | Fix |
|---------|-----|
| Backend won't start | Make sure `.venv` is activated and `.env` file exists |
| "Address already in use" | Run `lsof -ti:8000 \| xargs kill -9` to kill the old server |
| Frontend won't start | Run `npm install` first |
| Translation says "(mock)" | Your Gemini API key is missing or wrong in `.env` |
| No audio in response | Your ElevenLabs API key is missing or wrong in `.env` |
| CORS error in browser | Backend is not running, or wrong port |

---

## Quick Smoke Test Checklist

- [ ] `http://localhost:8000/api/health/` returns `{"status": "ok"}`
- [ ] `http://localhost:5173` shows the landing page
- [ ] `python test_ws.py` shows a real Spanish translation
- [ ] Audio is generated (non-empty `audio_base64`)
- [ ] Session end generates a medical summary
